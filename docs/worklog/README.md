# worklog

`_docs/worklog/` は、作業メモ・未解決課題・次の TODO を残すためのログ置き場です。  
並行開発（git worktree / 複数 agent）での認知負荷と手戻りを減らす目的で使います。

## ファイル命名規則
- feature ごと：`feature-A.md`, `feature-B.md`
- 追加で日付を付けてもよい：`feature-A-2026-01-02.md`

## 記載内容の例
以下のような形で、短くても良いので継続的に追記してください。

### 今日やったこと
- 例：API のエンドポイント案を作成

### 課題
- 例：既存 `_docs` との差分整理が必要

### 次やること
- 例：提案を `_docs/proposals/feature-A/` にまとめてレビュー依頼
